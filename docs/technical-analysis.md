# sensor-detect 技术分析

## 目标

构建一个本地可运行的 AI 生成图像检测网站。第一阶段重点不是账号、部署、在线访问，而是打通上传图片、后端检测、前端展示结果的闭环。

## MVP 范围

- 单图上传检测
- 返回 AI / Real 概率
- 展示检测模式、置信度、模型名称和风险提示
- 本地浏览器可访问
- 检测服务可替换，方便后续接入更强模型

## 技术选型

### 前端

第一版使用静态 HTML / CSS / JavaScript。原因是当前目标是快速本地跑通，不需要先引入 React、Vite、路由、状态管理等构建链。后续如果页面复杂起来，再迁移到 React + TypeScript。

### 后端

第一版使用 Python 标准库 HTTP 服务。原因是可以不依赖外部包先启动，避免本地环境、网络下载、深度学习依赖安装拖慢验证。

服务内部将检测逻辑封装为 `DetectorService`。未来切换 FastAPI、ONNX Runtime、PyTorch 或独立模型服务时，前端 API 不需要大改。

### 模型推理

默认模式：

- `demo`：本地无依赖也能运行，用图片基础特征生成结构化结果，仅用于链路验证。

可选真实模型模式：

- `model`：通过 Hugging Face `transformers` 的 `image-classification` pipeline 加载模型。
- 默认模型建议：`haywoodsloan/ai-image-detector-deploy`。

## 风险

AI 生成图像检测不是确定性事实判断。模型在训练集外的新生成器、二次压缩、裁剪、重采样、截图转存等场景下可能失效。因此产品表达应使用“概率”“风险”“可信度”，避免使用“真图”“假图”的绝对措辞。
